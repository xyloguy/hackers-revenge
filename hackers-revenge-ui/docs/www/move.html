HALT/CATCH FIRE

In HALT/CATCH FIRE your task is to take revenge on your fellow engineers by writing simple programs to defeat your opponent in a tiny, memory-based arena. Your programs will consist of a simple list of instructions running in a shared memory space with your opponent. You can defeat your opponent by tricking them into running an instruction that will cause it to error or simply catch fire.

As a hacker in HALT/CATCH FIRE you have a pallet of instructions in order to execute your strategy of engineering dominance. Your program runs in a memory space of 256 locations shared with your opponent. Your program will run against opponents in a battle of 4 rounds in which the system will alternate who goes first and randomize the starting position of each program. Each round will begin by loading your program and the opponent's into memory at a random position. Your program will consist of a simple list of a maximum of 25 instructions

Your program is stack-based which is your primary storage area for variables and data. The stack operates as a list and you can push numbers onto the top of the list and some instructions take items from the top of the list (a.k.a pop from the list), and some instructions pop from the top and push results back, like ADD which pops two numbers, adds them together and pushes the result to the top of the list. Your stack is limited to 16 items. If you attempt to push more than 16 items to the stack, or pop an item from an empty stack, your program will terminate and your opponent will win. The only data type in HALT/CATCH FIRE is a number that can be a maximum of 255 and a minimum of -255.

Your program also always accesses memory relative to its current position. For example, you cannot directly address memory address 0, but you can address a memory location that is 15 positions ahead or behind. You can jump forwards or backwards from your current position 3 but not to a specific memory address. This allows your program to operate the same regardless of where they are randomly placed in the arena.

  • Memory Space: 256 locations. Each location contains a whole instruction. Each memory location is initialized with NULL. If you attempt to execute a NULL your program will terminate.
  • The Stack: 16 slots, stack overflow and underflow will result in termination.
  • Programs: Programs can be up to 25 instructions. The instruction HCF (halt and catch fire) will cause your program to terminate (and catch fire) and JUMP 0 will also cause your program to catch fire.
  • Instructions: there are an assortment instructions available to you. Some instructions take an argument; all arguments are integers in range -255 to 255 inclusive.

Your program will run against each opponent in 12 battles. Each battle consists of 4 rounds and in each round the system alternates which program starts first and places each program in a random location. Programs run for a maximum number of 1000 cycles. If both programs are still running after 1000 cycles the round results in a tie.

A battle is won by surviving more rounds than your opponent. For each battle you win you will score 200 points, for a tie you will score 100 points and a loss will result in 0 points.

When your program is submitted the system will run your program against each program currently in the top ten and if your program scores higher than any in the top ten you will be positioned in the new top ten and the lowest scoring player will drop.

NOOP
No Operation. Do nothing.
PUSH n
Push value n to the top of the stack.
DROP
Remove the top item on the stack.
DUPE
Duplicate the top item on the stack.
DUPE2
Duplicate the top item on the stack twice.
SWAP
Swap the order of the top two items on the stack.
INC n
Add the value n to the item on the top of the stack.
ADD
Pop two items from the stack, add them and push the result back to the top.
SUBTRACT
Pop two items from the stack, subtract them and push the result back to the top.
DIVIDE n
Pop top item from the top of stack and divide it by n and push the result back to the stack. If n is 0, then the stack is unaffected
MULTIPLY n
Pop top item from the top of stack and multiply it by n and push the result back to the stack. If n is 0, then the stack is unaffected
GT n
Checks if item on top of stack is greater than n and puts 1 on stack if true, or 0 if false.
LT n
Checks if item on top of stack is less than n and puts 1 on stack if true, or 0 if false.
NEGATE
Negate the item at the top of the stack, e.g 1 becomes -1 and -1 becomes 1.
JUMP p
Jump to relative location p. e.g JUMP -1 jumps to the instruction before, JUMP 2 jumps two forward. JUMP 0 will cause your program to terminate.
JUMPZ p
Pop value from the top of the stack, jump to relative location p if the value is 0.
JUMPG p
Pop value from the top of the stack, jump to relative location p if the value is greater than 0.
HCF

Stop running and catch fire (you lose.)

Cluster Effect: if you copy a HCF on top of an existing HCF instruction, the fire will spread to the neighboring cells.

SCAN n
Pop a relative memory address from the top of the stack and scan n locations starting from that address. A number will be pushed to top of the stack depending on what is found:
  • -1: Found a HCF instruction.
  • 0: Nothing Found.
  • 1: Found another instruction
For example: PUSH 10 ; SCAN 5 will scan 5 locations, starting at 10 locations away. SCAN -10 will scan 10 locations backwards from the start position.

The way you defeat your opponent is by forcing them to execute an instruction that causes it to terminate. This can be done by forcing them to execute a bad instruction.

  • Catch Fire: force your opponent to halt and catch fire by moving a HCF instruction into its path.
  • Cluster Effect: if you copy a HCF on top of an existing HCF instruction, the fire will spread to the neighboring cells.
  • Scaredy Cats: a JUMP 0 instruction will cause your program to terminate.
  • Error: force your opponent to terminate by forcing an error, either by copying a NULL memory space into its path or copying instructions that make it overflow or underflow the stack.
  • Cliff-Jumping: cause your opponent to jump into NULL memory space.
  • Corruption: use the SETARG instruction to alter your opponent's program to cause errors.
  • Flooding: flood the memory space with bad instructions and hope you hit your opponent before you hit yourself.
  • Targeting: use the SCAN instruction to find your opponent and target them that way.
